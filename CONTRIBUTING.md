# Contributing to the Stytch Mobile SDK Monorepo

Thank you for your interest in contributing! We welcome contributions from the community.

All third party contributors acknowledge that any contributions they provide will be made under the same open source license that the open source project is provided under.

## How to Contribute

- **Bug Reports & Feature Requests:**

  - Open an issue describing the bug or feature request.
  - Please provide as much detail as possible, including steps to reproduce bugs.

- **Pull Requests:**
  - Fork the repository and create your branch from `main`.
  - Write clear, concise commit messages.
  - Add or update tests as appropriate.
  - Run `./build lint` and `./build test` before submitting.
  - Open a pull request and describe your changes.

## Coding Standards

- Use [KtLint](https://github.com/pinterest/ktlint) and the [ktlint gradle plugin](https://github.com/JLLeitschuh/ktlint-gradle) for code style and formatting.
- Write clear, maintainable, and well-documented code.
- Use descriptive variable and function names.
- Add comments where necessary.
- Add unit tests where reasonable.
